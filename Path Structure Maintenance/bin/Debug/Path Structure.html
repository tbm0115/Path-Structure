Path Structure

This document is not controlled. Created 06/20/2016 12:49 PM

Folders and Files with a name enclosed in {} are wildcard objects, meaning that the object name can be any valid object name.

  • Structure: \\server\Parts
    • {PartNo}: Contains all documents related to the specific Part #.
      • Print: A dimensional part drawing.
        • Master: The master, or most current, dimensional part drawing.
        • Revision: A revision of a dimensional part drawing that may be outdated, but still valid.
      • Setup Sheet URL: Provides a direct link to the part's associated setup sheet on the Setup Sheet Site
      • Archive: Contains outdated documents per QMSP-1002.
        • ArchivedFile: Archived file.
      • CAD-CAM: Contains specific part process CAD/CAM models and any other Computer Aided Engineering documents
        • CAD File: A CAD file of the part.
          • Master: The master, or most current, CAD model of the part.
          • Revision: A revision of a CAD model of the part.
        • CAM File: A CAM document for the specific part process.
          • Master: A master CAM document for the specific part process.
          • Revision: A revised CAM document for the specific part process.
        • Vericut: Contains support documents for the program Vericut.
          • Vericut Project File: The main Vericut project file.
          • Vericut Support File: A support document for the program Vericut.
        • {CAD-CAM Support Folder}: Contains any necessary support documents for CAD-CAM applications
          • Support File: A CAD-CAM support document.
      • Customer Originals: Contains the orginal documents provided by the customer. These files should not be replaced
        • Original Customer Document: An original customer document
      • Documents: Contains in-house support documents.
        • Setup Sheet: A document explaining procedural best practices.
        • In-House Drawing: An In-House dimensional part drawing. For internal use only
        • Part Render: A rendering of the part for use in various software as a thumbnail
      • Inspection: Contains quality inspection documents for the specific part.
        • {JobNo}: Contains order-specific, procedural documents.
          • FAIR: A First Artical Inspection Report for a specific part.
        • Inspection Program: A procedural part inspection routine.
          • MicroVu: A procedural part inspection routine for use on the Micro-Vu vision system.
          • Romer: A procedural part inspection routine for use on the Romer Arm.
          • MIDI: A procedural part inspection routine for use with the Manual Inspection Data Input application.
        • Inspection Data: An intermediate procedural database.
          • MicroVu: A temporary data storage file from the Micro-Vu vision system.
          • Romer: A temporary data storage file from the Romer Arm.
          • MIDI: A temporary data storage file from the Manual Inspection Data Input application.
        • Bubble Print: A dimensional part drawing specifying the indices of pertinent dimensions.
          • Master: The master, or most current, specification of pertinent dimensions.
          • Revision: A revision of the specification of pertinent dimensions.
      • NC Programs: Contains procedural part machining routines.
        • Part Program: A procedural part machining routine.
          • Fusion: A procedural part machining routine for use on a Fusion machining controller.
          • Matrix: A procedural part machining routine for use on a Matrix machining controller.
          • Matrix II: A procedural part machining routine for use on a Matrix II machining controller.
          • None: A procedural part machining routine for use on an unidentified machining controller.
      • Outside Processes: Contains all outside processes for the provided part
        • {OutsideProcess}: Contains various outside processing files/documentation
          • Outside Process Drawing: A dimensional part drawing of the outside process.
          • External File: Represents any external processing file
        • Quote Documents: Contains various documents required for quoting
          • Quote Document: Represents a quote document
    • Quality: Contains customer quality requirements, if any, per QMSD-1001.
      • Quality Requirements: A customer quality requirements document.
